# 参与贡献

感谢你愿意参与 MD阅读器。这个项目由 **谁是专家** 发起，原则是：轻量、清楚、离线优先，优先解决手机上高频阅读 Markdown 的真实问题。

## 适合提交的方向

- 修复 Android 机型兼容、文件打开、历史记录、WebView 渲染等问题。
- 改善移动端阅读体验，例如搜索、目录、代码块复制、长文跳转。
- 增强 Markdown 语法支持，但需要评估包体、性能和安全影响。
- 补充真实场景反馈，尤其是 AI 生成文档、聊天转发文件、知识库片段、代码说明等使用链路。

## 提交前建议

1. 尽量让改动小而聚焦。
2. 不要提交 APK、AAB、keystore、local.properties 或本地 IDE 缓存。
3. 如果改动涉及 Markdown 渲染，请说明测试过的语法样例。
4. 如果改动涉及“从其他应用打开”，请说明测试来源，例如微信、文件管理器、QQ、邮件。
5. 如果改动影响包体大小，请在 PR 中说明变化。

## 本地构建

```bash
./gradlew :app:assembleDebug
./gradlew :app:assembleRelease
```

release 构建在没有本地 keystore 时会回退到 debug 签名，便于开源协作环境构建验证。正式分发请使用自己的 keystore。

## 维护取舍

这个项目不会追求成为完整 Markdown 编辑器。功能进入主线前，优先考虑三个问题：

- 是否解决手机阅读 Markdown 的高频痛点？
- 是否保持应用足够轻量？
- 是否能在离线或弱网状态下稳定使用？
