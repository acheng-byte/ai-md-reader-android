# 路线图

MD阅读器由 **谁是专家** 发起，长期方向不是“做一个很大的编辑器”，而是做一个手机上足够顺手的 Markdown 阅读入口。尤其是在 AI 辅助编码和 AI 内容生成越来越日常之后，Markdown 会大量出现在聊天、文件、知识库、会议纪要和自动化产物里。

## 维护原则

- 轻量：新增功能不能明显拖慢启动、渲染或包体。
- 离线：核心阅读链路尽量不依赖网络。
- 移动端友好：优先解决小屏、单手、长文、从其他应用打开的体验。
- 安全：默认不执行不可信 Markdown 中的 HTML 或脚本。

## 已完成

- v1.0：本地打开、源码 / 预览切换、字号 / 行距 / 段距 / 明暗主题、从其他应用“用其他应用打开”。
- v1.1：文档目录跳转、标题折叠 / 展开、打开历史。
- v1.2：代码块一键复制、仅限 Markdown 选择、打开历史区分“授权过期 / 已删除”、收藏夹（复制副本、按文件去重、取消即删）。
- v1.3：收藏切换移入工具栏常显（矢量星标）、工具栏布局精简、显示设置改为点击屏幕中央唤出、标题折叠箭头优化。
- v1.3 iOS：新增 SwiftUI + WKWebView 原生 iOS 源码，功能对齐 Android v1.3，并完成模拟器构建、安装、启动、浅色 / 深色首屏冒烟。
- v1.4（Android / 鸿蒙兼容设备）：一键转发——把完整 `.md` 文档经系统分享转发到微信等社交应用（如微信好友）。
- v1.4.1（Android / 鸿蒙兼容设备）：阅读位置记忆；修复已收藏文档在打开历史中误报“授权过期”的问题。

## 近期方向

- 文内搜索：支持关键词高亮、上一个 / 下一个跳转。
- 长文阅读：进一步优化目录联动、滚动定位和章节级定位体验。
- 打开来源兼容：继续改善微信、文件管理器、QQ、邮件等入口。
- 收藏夹增强：排序、重命名、按来源分组。

## 中期方向

- 增强 Markdown 语法：任务列表、脚注、数学公式、Mermaid 等需要逐项评估。
- 分享与导出：将 Markdown 预览分享为图片、HTML 或 PDF。
- 多文件场景：围绕一个文件夹或知识库目录做轻量导航。
- AI 内容场景：优化长提示词、代码说明、会议纪要、项目方案等阅读模板。

## 暂不优先

- 完整 Markdown 编辑器能力。
- 复杂云同步。
- 重型插件系统。
- 需要持续联网的服务端能力。
