# 版本记录

## v2.1.7 - 2026-07-12

- 修复：PDF/DOC 文档点击编辑/源码切换按钮时崩溃 — 切换前检测二进制格式，弹出提示"不支持编辑"并阻止进入源码模式；`trySaveInPlace` 增加 MIME 类型兜底检测。
- 修复：TXT 文件行首数字前出现多余反斜杠 — 改用零宽空格（U+200B）插入数字与标点之间打断有序列表模式，不再显示可见反斜杠。

## v2.1.6 - 2026-07-12

- 修复：渲染前彻底清除替换字符（U+FFFD 菱形问号）—— 在 `normalizeText()` 中增加最终安全网，无论编码检测是否完美，菱形符号都不会出现在最终渲染结果中。
- 优化：编码检测阈值从 1% 降至 0.1%，更多边缘文件能触发 GB18030 比较回退。

## v2.1.5 - 2026-07-12

- 修复：编码检测逻辑优化 — 回退 GB18030 前先比较两种解码的替换字符数（），仅当 GB18030 确实更少时才采用，避免误切换导致空格变成菱形问号乱码。
- 修复：iOS 端同步更新编码比较逻辑。

## v2.1.4 - 2026-07-12

- 修复：编码检测增强 — 新增 UTF-16LE / UTF-16BE BOM 自动识别，回退编码改用 GB18030（GBK 超集，覆盖更多汉字），解决部分文件乱码问题。
- 修复：换行符统一 — `\r\n` 和孤立 `\r` 自动规范化为 `\n`，消除 Windows / Mac / Linux 来源文件的换行差异。
- 修复：TXT 纯文本渲染 — 补全 Markdown 特殊字符转义（`~`、`=`、`!`、`<`、`+`、行首数字），防止 `~~删除线~~`、`==高亮==`、有序列表等被误解析。
- 修复：特殊空格处理 — 全角空格（U+3000）、不间断空格（U+00A0）、零宽字符（ZWS/ZWJ/BOM 等）统一规范化，避免不可见字符导致排版异常。
- 修复：iOS 端编码支持 — `DocumentService` 新增 BOM 检测链和 GB18030 回退，与 Android 端对齐。

## v2.1.3 - 2026-07-12

- 新增：图片单击预览 — 点击内嵌图片（含外链）打开全屏预览，复用已有的双指缩放 + 单指拖动 + 双击重置能力。
- 新增：视频单击全屏预览 — 点击 `![[video]]` 嵌入的视频自动全屏播放，双击关闭。
- 新增：音频播放支持 — `![[audio.ext]]` 支持 mp3/wav/ogg/m4a/aac/flac/wma 格式，渲染为 HTML5 `<audio>` 播放器。
- 优化：图片光标变为 `zoom-in` 提示可点击预览。
- 优化：中心点击排除列表新增 `AUDIO` 标签及音频/视频容器，防止误触设置面板。
